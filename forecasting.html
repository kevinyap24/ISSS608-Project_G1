1 Introduction

Building on from the EDA section, this part proceeds to implement a time series forecasting workflow for Singapore Consumer Price Index (CPI).

CPI dataset used in this study covers monthly data from January 2020 to December 2025, structured across hierarchical classification of expenditures ranging from overall CPI down to detailed items. This hierarchical structure mandates forecast to be generated at the most granular level and when aggregated up it must remain consistent.

2 Methodology

The forecasting workflow uses modeltime and *tidymodels** frameworks. These framework provides a unified interface for training, evaluating, and refitting time series models. These frameworks also support both classical statistical models and modern machine learning approaches.

Each of the CPI series are evaluated using the following forecasting models:

Model Type Key Strength
ETS Classical Captures trend and seasonality via exponential smoothing
Auto-ARIMA Classical Automated order selection for optimal fit on stationary series
Auto-ARIMA + XGBoost Hybrid Combines Auto-ARIMA with XGBoost to capture non-linear residuals
Prophet Additive Regression Robust to missing data and handles seasonal shifts flexibly

Models are trained and evaluated using nested forecasting workflow and through modeltime_nested_fit() function. Each CPI leaf nodes are treated as independent nested dataset, with the last 12 months held out as a test set. The model with the lowest RMSE on the test set is selected as the best model for the particular serie. All selected models are then refitted on the full data before generating the 12-month forecast.

After generating leaf-level forecasts, weighted bottom-up aggregation is applied. Parent-level forecasts are computed as the weighted average of their children forecasts, using the official DOS CPI basket weights. This process is repeated from the most granular level up to the overall CPI. Same aggregation is applied to the 90%, 95%, and 99% Confidence Interval.

3 Time Series Forecasting Workflow

The forecasting workflow is structured into the following stages:

  1. Data Preparation begins by loading the clean dataset produced from the EDA part. The data is padded to ensure complete monthly time series for every series, and missing values (if any) are imputed using linear interpolation using ts_impute_vec().

  2. Hierarchy Derivation extracts the parent-child relationships. Leaf nodes are identified as series that never appear as a parent of any other series in the dataset.

  3. Nested model Training filters the data to leaf nodes and trains the four models on the leaf nodes only across all CPI using modeltime_nested_fit(). The best model is selected based on the minimal RMSE on the 12-month holdout test set.

  4. Refitting and Forecasting refits each selected model on the full dataset and generates 12 month foreacast at three different Confidence Intervals: 90%, 95%, 99%.

  5. Bottom-up Aggregation combines leaf-level forecasts into parent-level forecasts using official expenditure weights defined by DOS.

5 Data Preparation

5.1 Padding and Imputing Missing Values

The data is grouped by series and padded to ensure complete monthly sequence for every CPI category. Even though this dataset has been vetted for nulls, for reusability, any gaps introduced during padding, if the dataset contains nulls or inconsistencies, are filled using linear interpolation via ts_impute_vec() with a period of 1, followed by a forward and backward fill as a safety net for any remaining edge cases.

cpi_data_long %>% filter(is.na(cpi))
# A tibble: 0 × 7
# ℹ 7 variables: date <date>, series <chr>, cpi <dbl>, level <dbl>,
#   division <chr>, group <chr>, class <chr>
cpi_data_long <- cpi_data_long %>%
  group_by(series) %>%
  arrange(date) %>%
  pad_by_time(.date_var = date, .by = "month") %>%
  mutate(cpi = ts_impute_vec(cpi, period = 1)) %>%
  fill(cpi, .direction = "downup") %>%
  ungroup()

5.2 Deriving Parent Lookup Table

Rather than defining the parent-child relationships manually, the relationships are to be defined directly from the dataset’s existing hierarchy columns. Each series is assigned a parent based on its level

Level 1 series consolidate to the overall CPI. Level 2 series report to their division. Level 3 report to their group.

parent_lookup <- cpi_data_long %>%
  distinct(series, level, division, group) %>%
  mutate(Parent = case_when(
    level == 1 ~ "Consumer Price Index (CPI)",
    level == 2 ~ division,
    level == 3 ~ group,
    TRUE ~ NA_character_
  )) %>%
  select(series, Level = level, Parent)

5.3 Identifying Leaf Nodes (Series with No children)

Leaf nodes are series that never appear as a parent of any other series. This approach handles mixed-depth of the CPI hierarchy, where Food series extend to level 3 while other branches are at level 2.

all_parents <- parent_lookup %>%
  filter(!is.na(Parent)) %>%
  pull(Parent) %>%
  unique()

leaf_categories <- cpi_data_long %>%
  filter(level > 0,
         !series %in% all_parents) %>%
  distinct(series) %>%
  pull(series)

6 Nested Forecasting

6.1 Preparing Nested Forecasting Data

The data is filtered to leaf nodes only and structured into a nested time series format using nest_timeseries(). Each series is extended by 12 months to create placeholders for forecasting data. The last 12 months of observation data is reserved as hold-out period data (test set) using split_nested_timeseries().

nested_data_tbl <- cpi_data_long %>%
  filter(series %in% leaf_categories) %>%
  extend_timeseries(
    .id_var = series,
    .date_var = date,
    .length_future = 12
  ) %>%
  nest_timeseries(
    .id_var = series,
    .length_future = 12
  ) %>%
  split_nested_timeseries(.length_test = 12)

6.2 Defining Models

Four models are specified as candidates fore each CPI series.

ETS and ARIMA are classical statistical models that are suited for short monthly time series with clear trend and seasonal patterns. ARIMA boost is an enhanced version of ARIMA by adding gradient-boosted tree features using machine learning, allowing it to capture non-linear patterns. Prophet decomposes series into trends, seasonality, and holiday effects.

model_ets     <- exp_smoothing() %>% set_engine("ets")
model_arima   <- arima_reg()     %>% set_engine("auto_arima")
model_boost   <- arima_boost()   %>% set_engine("auto_arima_xgboost")
model_prophet <- prophet_reg()   %>% set_engine("prophet")

6.3 Building Model Workflows

Each model is wrapped in a workflow() paired with a recipe. The ARIMA-Boost workflow includes additional preprocessing steps to extract time series features such as month, quarter, and year indicators, which serve as predictors for the XGBoost component.

wf_ets <- workflow() %>%
  add_model(model_ets) %>%
  add_recipe(recipe(cpi ~ date, data = extract_nested_train_split(nested_data_tbl)))

wf_arima <- workflow() %>%
  add_model(model_arima) %>%
  add_recipe(recipe(cpi ~ date, data = extract_nested_train_split(nested_data_tbl)))

wf_prophet <- workflow() %>%
  add_model(model_prophet) %>%
  add_recipe(recipe(cpi ~ date, data = extract_nested_train_split(nested_data_tbl)))

wf_boost <- workflow() %>%
  add_model(model_boost) %>%
  add_recipe(
    recipe(cpi ~ date, data = extract_nested_train_split(nested_data_tbl)) %>%
      step_timeseries_signature(date) %>%
      step_rm(matches("(xts$)|(iso$)|(hour)|(minute)|(second)|(am.pm)")) %>%
      step_dummy(all_nominal_predictors(), one_hot = TRUE)
  )

6.4 Fitting Models

All four models above are fitted simultaneously across all leaf-nodes using model_nested_fit().

nested_modeltime_tbl <- modeltime_nested_fit(
  nested_data = nested_data_tbl,
  wf_ets, wf_arima, wf_boost, wf_prophet,
  control = control_nested_fit(allow_par = FALSE, verbose = TRUE)
)

6.5 Selecting the Best Model

The best performing model for each series is selected based on the lowest Root Mean Squared Error (RMSE) on the test data. This ensures that the model used to generate forecast data has demonstrated accuracy to unseen data.

best_nested_models <- nested_modeltime_tbl %>%
  modeltime_nested_select_best(
    metric                = "rmse",
    minimize              = TRUE,
    filter_test_forecasts = TRUE
  )

6.6 Refitting and Forecasting Leaf Nodes

Models selected as the best models are refitted to the full dataset, including the hold-out period data, before generating the 12-month forecast data. Forecasts are generated at three Confidence Interval levels of 90%, 95%, and 99% by running refitted_nested_models() three times for each of the Confidence Intervals and joining the resulting CI into a single table.

refitted_nested_models <- best_nested_models %>%
  modeltime_nested_refit(control = control_nested_refit(verbose = TRUE))
leaf_fc_90 <- refitted_nested_models %>%
  modeltime_nested_forecast(
    h = 12,
    conf_interval = 0.90,
    control = control_nested_forecast(verbose = TRUE)
  ) %>%
  filter(.key == "prediction") %>%
  select(date = .index, series, .value,
         lo90 = .conf_lo, hi90 = .conf_hi)
leaf_fc_95 <- refitted_nested_models %>%
  modeltime_nested_forecast(
    h = 12,
    conf_interval = 0.95,
    control = control_nested_forecast(verbose = TRUE)
  ) %>%
  filter(.key == "prediction") %>%
  select(date = .index, series, lo95 = .conf_lo, hi95 = .conf_hi)
leaf_fc_99 <- refitted_nested_models %>%
  modeltime_nested_forecast(
    h = 12,
    conf_interval = 0.99,
    control = control_nested_forecast(verbose = TRUE)
  ) %>%
  filter(.key == "prediction") %>%
  select(date = .index, series, lo99 = .conf_lo, hi99 = .conf_hi)
leaf_fc <- leaf_fc_90 %>%
  left_join(leaf_fc_95, by = c("date", "series")) %>%
  left_join(leaf_fc_99, by = c("date", "series"))

6.8 Bottom-up Aggregation

The aggregate_bottom_up() function iterates through the hierarchy from the deepest level upward. At each level, child forecasts are joined to their respective parent using the parent_lookup table, and the parent’s forecast is computed as the expenditure-weighted average of its children’s forecasts. Both the point forecast and all confidence interval bounds are aggregated using the same weights.

aggregate_bottom_up <- function(forecasts, parent_lookup, weights_tbl) {
  all_levels <- forecasts %>%
    left_join(weights_tbl, by = "series")

  levels_to_agg <- parent_lookup %>%
    filter(!is.na(Parent)) %>%
    arrange(desc(Level)) %>%
    pull(Level) %>%
    unique()

  for (lvl in levels_to_agg) {
    child_nodes <- parent_lookup %>%
      filter(Level == lvl) %>%
      select(series, Parent)

    parent_fc <- all_levels %>%
      inner_join(child_nodes, by = "series") %>%
      group_by(date, series = Parent) %>%
      summarise(
        .value = sum(.value * Weight) / sum(Weight),
        lo90   = sum(lo90   * Weight) / sum(Weight),  
        hi90   = sum(hi90   * Weight) / sum(Weight),  
        lo95   = sum(lo95   * Weight) / sum(Weight),  
        hi95   = sum(hi95   * Weight) / sum(Weight),  
        lo99   = sum(lo99   * Weight) / sum(Weight),  
        hi99   = sum(hi99   * Weight) / sum(Weight),  
        Weight = sum(Weight),
        .groups = "drop"
      )

    all_levels <- bind_rows(all_levels, parent_fc) %>%
      distinct(date, series, .keep_all = TRUE)
  }
  return(all_levels)
}
